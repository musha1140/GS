How it works

The cipher

The song is the key. A MIDI file is a constant, so its parsed note sequence is a constant too. Every note contributes three bytes (its number, velocity, and ticks) into a deterministic keystream. Your input, a typed message or any file's raw bytes, is XORed against that keystream, cycling circularly when the lengths differ, which they always do. XOR twice with the same song and you get the original back, bit for bit. It is a lossless chop.

Two things reconstruct the message: the .slayy file and the song. An AES-256-GCM seal (Web Crypto, PBKDF2) wraps the payload so decompiling verifies integrity, and the decompiler re-derives the keystream from the song to prove the bits check out. The passphrase lives in this page's source: this is art, not security.

Send a message via zero width

Zero width characters are real Unicode code points that render as nothing: the zero width space (U+200B) and zero width non joiner (U+200C). Map bit 0 to one and bit 1 to the other and you can carry a whole hidden payload inside ordinary looking text. Paste a cover word, append the invisible run, and the message rides along unseen in any chat box or caption.

Encrypt a message with a song first, Happy Violence or any of them, then wrap the resulting share link as hidden text below. Whoever receives it reveals the link, and the same self decoding transmission plays out. The deeper tool and the write up live in the repo: steganographr.
